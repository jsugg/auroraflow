# Architecture Decision Log

This log records stakeholder decisions that gate AuroraFlow architecture implementation work. It is the durable source for `AUR-DEC-*` policy; supporting traceability keys live in [Architecture and QE Traceability Registry](./traceability.md).

## Maintenance rules

- Keep one row per `AUR-DEC-*` decision.
- Update this file before a behavior-changing PR relies on a changed decision.
- Link follow-up work with ADRs, task-owned docs, validation evidence, or traceability keys where applicable.
- Record implementation progress and validation output in PR evidence, ADRs, task-owned docs, or the traceability registry.
- Revisit decisions only with new evidence listed in the table; do not silently broaden scope.
